## 2. Functional Requirements

### 2.1 Visitor Management

#### FR-VIS-001: Track New Visitor
**Priority:** High  
**Description:** System shall create new visitor record when unknown visitor accesses tracked pages.

| Field | Type | Required | Validation |
|-------|------|----------|------------|
| visitor_id | VARCHAR(100) | Yes | Unique, generated by tracking script |
| ip_address | VARCHAR(45) | No | Valid IPv4/IPv6 |
| user_agent | TEXT | No | Browser user agent string |
| first_visit | DATETIME | Yes | Current timestamp |
| last_visit | DATETIME | Yes | Current timestamp |
| visit_count | INT | Yes | Default 1 |

**Acceptance Criteria:**
- Visitor record created with unique visitor_id
- Timestamps set to current time
- visit_count = 1

---

#### FR-VIS-002: Update Existing Visitor
**Priority:** High  
**Description:** System shall update visitor record on subsequent visits.

**Business Rules:**
- If visitor_id exists, update last_visit
- Increment visit_count by 1
- Update user_agent if significantly different

**Acceptance Criteria:**
- last_visit updated to current timestamp
- visit_count incremented
- No duplicate visitor_id records

---

#### FR-VIS-003: Link Visitor to Customer
**Priority:** High  
**Description:** System shall associate visitor with customer record when identified.

**Input:** visitor_id, debtor_no

**Business Rules:**
- debtor_no must exist in debtors_master
- Link is permanent (one visitor, one customer)

**Acceptance Criteria:**
- debtor_no stored in visitor record
- Can query all visitors for customer

---

#### FR-VIS-004: Get Visitor Details
**Priority:** High  
**Description:** System shall retrieve visitor details by visitor_id.

**Input:** visitor_id (VARCHAR)

**Output:** Visitor object including:
- All visitor fields
- Customer name if linked
- Event count

**Acceptance Criteria:**
- Returns complete visitor data
- Returns null for non-existent visitor_id

---

#### FR-VIS-005: List Visitors
**Priority:** Medium  
**Description:** System shall list visitors with filtering.

**Input Parameters:**
| Filter | Type | Description |
|--------|------|-------------|
| debtor_no | VARCHAR | Filter by linked customer |
| date_from | DATETIME | Filter by first_visit |
| date_to | DATETIME | Filter by first_visit |

**Output:** Array of visitor objects

**Acceptance Criteria:**
- Filters work correctly
- Results include event counts

---

### 2.2 Event Tracking

#### FR-EVT-001: Record Event
**Priority:** High  
**Description:** System shall record tracking events with full context.

| Field | Type | Required | Validation |
|-------|------|----------|------------|
| visitor_id | VARCHAR(100) | Yes | Must exist in visitors |
| event_type | VARCHAR(50) | Yes | Predefined types |
| event_data | TEXT | No | JSON or text payload |
| page_url | VARCHAR(500) | No | Valid URL |
| referrer | VARCHAR(500) | No | Valid URL |
| ip_address | VARCHAR(45) | No | Valid IP |
| user_agent | VARCHAR(255) | No | Browser string |
| contact_id | INT | No | Logged-in user |
| event_time | DATETIME | Yes | Current timestamp |

**Event Types:**
- page_view
- click
- form_submit
- download
- video_play
- custom

**Acceptance Criteria:**
- Event created with all provided data
- visitor_id link verified
- event_time set to current timestamp

---

#### FR-EVT-002: Get Events by Visitor
**Priority:** High  
**Description:** System shall retrieve all events for a visitor.

**Input:** visitor_id (VARCHAR)

**Output:** Array of event objects ordered by event_time DESC

**Acceptance Criteria:**
- All events returned for visitor
- Ordered by time (newest first)
- Empty array if no events

---

#### FR-EVT-003: Filter Events by Type
**Priority:** Medium  
**Description:** System shall filter events by event_type.

**Input:** event_type (VARCHAR)

**Output:** Array of events matching type

**Acceptance Criteria:**
- Only matching type returned
- Can combine with visitor filter

---

#### FR-EVT-004: Filter Events by Date Range
**Priority:** Medium  
**Description:** System shall filter events by date range.

**Input:**
| Parameter | Type |
|-----------|------|
| date_from | DATETIME |
| date_to | DATETIME |

**Output:** Array of events within range

**Acceptance Criteria:**
- Date range filter works
- Inclusive of boundaries

---

#### FR-EVT-005: Get Event Count
**Priority:** Medium  
**Description:** System shall return event count for criteria.

**Input:** visitor_id or event_type or date_range

**Output:** Integer count

**Acceptance Criteria:**
- Accurate count returned
- Filters work correctly

---

### 2.3 Analytics

#### FR-ANA-001: Get Visitor Statistics
**Priority:** High  
**Description:** System shall return aggregated visitor statistics.

**Output:**
- total_visitors (last 30 days)
- total_events (last 30 days)
- unique_visitors

**Acceptance Criteria:**
- Statistics calculated correctly
- Date range configurable

---

#### FR-ANA-002: Get Event Type Statistics
**Priority:** Medium  
**Description:** System shall return event counts by type.

**Output:** Array of [event_type => count]

**Acceptance Criteria:**
- All event types included
- Counts accurate

---

#### FR-ANA-003: Get Page View Statistics
**Priority:** Low  
**Description:** System shall return most viewed pages.

**Output:** Array of [page_url => view_count]

**Acceptance Criteria:**
- Top pages ranked by views
- Configurable limit

---

### 2.4 CRM Integration

#### FR-CRM-001: Link Event to Contact
**Priority:** High  
**Description:** System shall link events to logged-in FA users.

**Business Rules:**
- If user logged into FA, capture contact_id
- Link events to customer if visitor linked

**Acceptance Criteria:**
- contact_id captured from session
- Events queryable by contact

---

#### FR-CRM-002: Customer Activity Summary
**Priority:** Medium  
**Description:** System shall return customer website activity.

**Input:** debtor_no

**Output:**
- Visit count
- Last visit date
- Event count by type
- Top pages visited

**Acceptance Criteria:**
- Activity summary for customer
- Empty if no activity

---

## 3. Data Validation Rules

### 3.1 Input Validation

| Field | Rule | Error Message |
|-------|------|---------------|
| visitor_id | Non-empty, max 100 chars | "Invalid visitor ID" |
| event_type | Must be in allowed list | "Invalid event type" |
| ip_address | Valid IP format | "Invalid IP address" |
| page_url | Valid URL format | "Invalid URL" |

### 3.2 Business Rules

| Rule | Condition | Action |
|------|-----------|--------|
| Visitor exists | visitor_id not found | Auto-create visitor |
| Customer link | debtor_no not found | Reject link |
